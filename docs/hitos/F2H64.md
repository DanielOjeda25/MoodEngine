## Qué entregó

### Bloque A — Framebuffer OIT separado

Nueva clase [`OpenGLOitFramebuffer`](../../src/engine/render/backend/opengl/OpenGLOitFramebuffer.h) (autónoma, ~60 LOC header + ~110 LOC cpp). Dos color attachments:

- `accumColor` RGBA16F (loc 0) — acumula `sum(rgb * α * weight)` en RGB y `sum(α * weight)` en A.
- `revealage` R16F (loc 1) — acumula `product(1 - α)` por blending multiplicativo desde el clear inicial `1.0`.

Depth attachment **compartido** con el `m_sceneFb` via `glFramebufferTexture2D` con texture id externo. El caller (SceneRenderer) es dueño del depth; este FBO solo lo referencia. Helper `ensureOitFb(w, h)` idempotente en `SceneRenderer` re-attachea el depth en cada resize (porque el scene FB puede haber recreado su depth texture).

### Bloque B — Shader OIT (uniform branching)

`pbr.frag` + `pbr_graph_template.frag` ganan uniform `int uOitPass`. Cuando `uOitPass==1`:

```glsl
float depth  = gl_FragCoord.z;
float weight = clamp(
    pow(min(1.0, effectiveOpacity * 10.0) + 0.01, 3.0)
        * 1e8 * pow(1.0 - depth * 0.9, 3.0),
    1e-2, 3e3);
FragColor = vec4(colorOut * effectiveOpacity, effectiveOpacity) * weight;
NormalRT  = vec4(effectiveOpacity);   // R16F: solo .r se escribe
```

El path forward F2H63 (`uOitPass==0` default) sigue funcionando inalterado. Path opaque NUNCA entra con `uOitPass==1` (el renderer no rutea opacos por el FB OIT).

### Bloque C — Composite shader

Nuevo [`oit_composite.frag`](../../shaders/oit_composite.frag) (~50 LOC, fullscreen tri). Divide `accum.rgb / max(accum.a, 1e-5)` para recuperar el color premultiplied. Output `vec4(averageColor, 1.0 - revealage)`. El caller compone sobre el scene FB con `glBlendFunc(GL_ONE_MINUS_SRC_ALPHA, GL_SRC_ALPHA)`. Reusa `post_process.vert`.

### Bloque D — PBR::oitPass reemplaza translucentPass de F2H63

`SceneRenderer_Render.cpp:791-880` ahora corre `PBR::oitPass` **sin sort manual** (el punto del hito):

1. Blit del scene color al backbuffer copy (F2H63 reusa esto para refracción).
2. Bind `m_oitAccumFb`. Clear separado por attachment via `glClearBufferfv` (accum=0, revealage=1).
3. Blend separado per-attachment (GL 4.0+):
   - loc 0 (accum): `glBlendFunci(0, GL_ONE, GL_ONE)` — suma pura.
   - loc 1 (revealage): `glBlendFunci(1, GL_ZERO, GL_ONE_MINUS_SRC_COLOR)` — multiplicativo.
4. `depthMask(GL_FALSE)`, `depthTest(GL_TRUE)`. Iterar `translucents` SIN sort.
5. Per-entity: `setInt("uOitPass", 1)`, bind backbuffer copy unit 7, draw.
6. Composite: bind `m_sceneFb` con `glDrawBuffers(1, GL_COLOR_ATTACHMENT0)` (preserva normales de los opacos en loc 1). Blend final + fullscreen tri.

### Bloque E — Shadow atlas RGB (sombras tintadas)

Cambio más invasivo del hito. Extiende `OpenGLShadowMapArray` con un **color attachment RGBA8 paralelo** a la depth array (mismo size + cascade count). `bindLayerAsTarget(layer, withColor=true)` re-attachea ambos via `glFramebufferTextureLayer`.

`ShadowPass::recordCsm` ahora hace **2 sub-pases por cascada**:

- **Opaque** (depth-only, `drawBuffer=NONE`): filtra `blendMode != Opaque` (vidrios no escriben depth — la luz pasa). Aditivos también filtrados.
- **Tinted** (nuevo): drawBuffer=COLOR_ATTACHMENT0. Itera Translucent con `castTranslucentShadow=true`. depth-write OFF + depth-test ON (vidrios detrás de un opaco no contribuyen). Blend `ZERO/SRC_COLOR` multiplicativo. El shader [`shadow_tinted.frag`](../../shaders/shadow_tinted.frag) emite `tint = albedoTint * (1 - opacity)`.

Color attachment se clearea a `(1,1,1,1)` por cascada → "luz blanca pasa" baseline. Cada vidrio multiplica el pixel actual → tintes superpuestos se acumulan (vidrio rojo + vidrio azul → sombra magenta tenue).

`pbr.frag::sampleShadow` cambia de `float` a `vec4` (factor + tint.rgb). La luz directional se multiplica por `shadowF * shadowTint`. Bind del color texture unit 8 (`uShadowColorMap`).

### Bloque F — UI checkbox + persistencia

`MaterialAsset.castTranslucentShadow = true` (default ON). Inspector Blending header gana checkbox **"Proyectar sombra tintada"** (es) / **"Cast tinted shadow"** (en), disabled si BlendMode != Translucent. Undo via `pushEditIfDone<u32>` (cast bool→u32, `InspectorEditTracker` variant no incluye bool).

Persistencia aditiva en `.moodmat`: key `cast_translucent_shadow` solo se escribe cuando `blendMode == Translucent && castTranslucentShadow == false`. Los `.moodmat` históricos cargan con default true sin necesidad de migración.

## Decisiones clave (detalle en `DECISIONS.md`)

1. **OIT reemplaza el sort manual de F2H63**. El path order-independent es el único; no se mantiene fallback con sort. Una sola forma de hacer las cosas.
2. **Uniform branching `uOitPass` sobre compilar 2 variantes del shader**. Simplicidad runtime. La rama OIT solo se ejecuta en translucents (que son pocos), el costo del branch es despreciable.
3. **FB OIT comparte depth con el scene FB**. Evita blit o resolve adicional. Depth-test funciona automáticamente contra opacos. Re-attach del depth en cada `resize`.
4. **Shadow atlas RGB con sub-pase tinted separado** (no atalpha-test en el opaque). Razón: vidrios no deben ocluir luz (físicamente correcto). El opaque pass filtra Translucent; el tinted pass agrega tinte sin tocar depth.
5. **Default ON para `castTranslucentShadow`** (opt-out, no opt-in). Vidrios nuevos tintan sombra automáticamente; el escape hatch existe para escenas con muchos translucents.
6. **OIT compatible con shader graph desde día 1**. El template `pbr_graph_template.frag` también emite ambos paths. Samples (water/glass/hologram) funcionan sin migración.
7. **Composite pass escribe solo a loc 0 del scene FB** (`glDrawBuffers(1, ...)`). Preserva normales de los opacos en loc 1 (SSR no se afecta).

## Limitaciones conocidas v1 (agendadas a futuros hitos)

- **Brushes translúcidos NO entran al OIT pass**. `RenderBatching` itera solo MeshRenderer. Brushes con material Translucent se siguen rendeando en el opaque path con depth-write ON (artefactos visuales). Workaround: convertir brush a mesh. **Agendado como F2H65 — Brushes translúcidos OIT** (~2-3h).
- **Sombras de Additive** — no implementadas (sin sentido físico).
- **Multi-bounce refraction** — un vidrio detrás de otro no refracta al primero correctamente (backbuffer copy es snapshot pre-OIT).
- **Performance per vidrio** — cada Translucent con `castTranslucentShadow=true` agrega un draw call por cascada al shadow pass. Con 50+ vidrios y 4 cascadas se puede notar. Mitigación: opt-out per material.
- **HDRI fijo** — el skybox `sky_kloofendal.png` es luminoso y dificulta visualizar sombras tintadas tenues. Workaround: bajar exposure del Environment. **Agendado como F2H66 — Skybox / HDRI switcher** (~2-3h).
- **Shift+D duplicate** — feature pedida durante validación. **Agendada como F2H64.1** (~30 min).
- **GL perf warning "REDUNDANT_FBO"** — el ShadowPass cambia drawBuffer entre opaque y tinted sub-pases varias veces por cascada. No afecta funcionalidad. Optimización: eliminar bind redundante en cleanup futuro.
