## Qué entregó

### Bloque A — PhysicsWorld API constraint

Tres factories nuevas en [`PhysicsWorld.h`](../../src/engine/physics/world/PhysicsWorld.h):

```cpp
u32 createHingeConstraint(u32 bodyA, u32 bodyB, const glm::vec3& pivotWorld,
                          const glm::vec3& axisWorld, f32 limitMinDeg, f32 limitMaxDeg);
u32 createDistanceConstraint(u32 bodyA, u32 bodyB,
                             const glm::vec3& pivotA_world, const glm::vec3& pivotB_world,
                             f32 minDistance, f32 maxDistance);
u32 createPointConstraint(u32 bodyA, u32 bodyB, const glm::vec3& pivotWorld);
void destroyConstraint(u32 constraintId);
u32  constraintCount() const;
```

Storage interno: `std::unordered_map<u32, JPH::Ref<JPH::Constraint>>` + `nextConstraintId = 1` (id == 0 == "no constraint"). Uso de `JPH::BodyLockMultiWrite` para acceder bodies A y B sin race en el constraint setup. Hinge construye basis ortonormal desde `axisWorld` (fallback Z si paralelo a X).

### Bloque B — JointComponent

Componente nuevo en [`Components.h`](../../src/engine/scene/components/Components.h):

```cpp
struct JointComponent {
    enum class Type : u8 { Hinge = 0, Distance = 1, Point = 2 };
    Type      type = Type::Hinge;
    u32       targetEntity = kJointNoTarget;  // raw entt handle del body B
    glm::vec3 pivotLocal{0.0f};               // local space de A
    glm::vec3 axisLocal{0.0f, 1.0f, 0.0f};    // solo Hinge
    f32       limitMinDeg = -180.0f;           // solo Hinge
    f32       limitMaxDeg =  180.0f;
    f32       minDistance = 0.0f;              // solo Distance
    f32       maxDistance = 1.0f;
    // Runtime (NO persiste):
    u32  constraintId = 0;
    bool dirty = true;
};
```

**Decisión arquitectural**: un solo `pivotLocal` (asumido en local de A). Body B usa su origen como segundo pivot. Si el dev necesita offset en B, mueve el Transform de B. Simplifica la UX (un solo DragFloat3 en el Inspector); el caso multi-pivot queda para un hito futuro si emerge presión.

**`kJointNoTarget = UINT32_MAX`** — sentinel "sin asignar". El default `0` colisionaba con la primera entity creada en una scene fresca (raw `entt::entity{0}`), descubierto al fallar el test de Bloque E. Constante exportada desde Components.h para que todos los call-sites (PhysicsSystem materialize, Inspector check, SceneLoader resolve) usen el mismo símbolo.

### Bloque E — Persistencia `.moodmap`

Nuevo struct [`SavedJoint`](../../src/engine/scene/serialization/SceneSerializer.h) en `SceneSerializer.h` + campo `std::optional<SavedJoint> joint` en `SavedEntity`. El target body B se persiste por **TAG** (string), no por handle, porque los entt handles no son estables entre sesiones — mismo patrón paths-no-ids del Animator/Inventory.

- `serializeEntityToJson`: emite `"joint"` cuando la entity tiene `JointComponent`, resolviendo `targetEntity` → tag via `entity.scene()->entityFromHandle(handle)`.
- `parseEntityFromJson`: parsea `"joint"` aditivamente (mapas pre-F2H65 sin el campo se leen sin `JointComponent`).
- `SceneLoader::applyOneEntity`: materializa `JointComponent` + **eager tag lookup** (sirve para single-entity restore como undo de delete).
- `SceneLoader::applyEntitiesToScene`: **segunda pasada** post-creación que resuelve cualquier joint cuyo owner se creó antes que el target en la iteración. Si el tag no existe → log warn + `targetEntity = kJointNoTarget` y el dev re-asigna desde el Inspector.

## Bugs cazados durante implementación + fixes

1. **`glm::radians` missing** (Bloque A) — agregado `#include <glm/trigonometric.hpp>`.
2. **`JPH::BodyLockMultiWrite` missing** (Bloque A) — agregado `#include <Jolt/Physics/Body/BodyLockMulti.h>`.
3. **`PhysicsWorld` no copyable** (Bloque H) — refactor inicial tenía `TwoBodies { PhysicsWorld; u32 a; u32 b; }` para reusar bodies entre tests; deleted copy ctor lo bloqueó. Fix: cada test inline su propio `PhysicsWorld + createBody` (penalty de ~3 líneas por test, gain en aislamiento).
4. **`IShader::setFloat3` no existe** — primer intento usó esa firma, el API real es `setVec3`.
5. **Sentinel `targetEntity == 0` colisiona con primera entity** (Bloque E, **bug de diseño**) — descubierto cuando el primer test de round-trip falló con `targetTag == ""` después de crear "Marco" → "Puerta" en orden. La primera entity creada en una Scene fresca tiene raw `entt::entity{0}`, que casteado a u32 = 0. El check `targetEntity != 0` filtraba el target legítimo. Fix: introducir `kJointNoTarget = UINT32_MAX` (== `static_cast<u32>(entt::null)`) como sentinel, actualizar **5 call-sites** atomicamente.

## Decisiones clave (detalle en `DECISIONS.md`)

1. **Un solo `pivotLocal` (en local de A)**. Body B usa su origen. Simplifica UX (un DragFloat3 + un drop target). El caso multi-pivot queda diferido si emerge presión.
2. **Target body B obligatorio** (no auto-spawn). El joint en una entity sin target válido queda inerte (PhysicsSystem skipea materialización). Force el flujo "primero spawn B, después linkea". Evita estados intermedios confusos.
3. **Persistencia por TAG, no por handle**. Mismo patrón que paths-no-ids del Animator/Inventory. Handles entt no son estables entre sesiones; tags son user-facing. **Limitación conocida**: tags no son únicos (el primer match gana); collision = `forEach` retorna el primero, log info si hay ambigüedad.
4. **Sentinel `kJointNoTarget = UINT32_MAX`** sobre `0`. Bug de diseño descubierto en Bloque E: raw entt handle 0 es válido (primera entity creada). Constante exportada desde `Components.h`.
5. **Tag resolution en 2 pasadas (eager + post-load)**. `applyOneEntity` hace lookup inmediato (suficiente para undo); `applyEntitiesToScene` hace segunda pasada por si el owner se creó antes que el target.
6. **Drag-drop entity payload `MOOD_ENTITY`** desde HierarchyPanel a Inspector. Convención reusable para futuros componentes que linkeen entities entre sí (ej. parent-child, follow-target). Carga `entt::entity` (4 bytes) como payload binario; el drop target memcpy-extrae + valida vía `Scene::entityFromHandle`.
7. **Debug overlay solo bajo F1**. No es visualización permanente — alinear con el patrón de tile AABBs + trigger OBBs existentes. Toggle único cubre todos los debug overlays.

## Limitaciones conocidas v1 (agendadas a futuros hitos)

- **Multi-pivot (offset propio en B)** — diferido. Workaround: mover Transform de B.
- **Slider y Fixed constraint** — no implementados en F2H65 (solo Hinge/Distance/Point). Plan original F2H23 los menciona; quedan para sub-hito siguiente si el dev los necesita para gameplay.
- **Vehicle constraint** — Jolt tiene `VehicleConstraint` nativo, pero out of scope F2H65. Plan original F2H25.
- **Ragdoll auto-build** — out of scope F2H65 (esqueleto + bodies + constraints automáticos). Plan original F2H24.
- **Cleanup en Play→Editor transition** — el constraint se mantiene entre frames mientras la entity esté viva. Si el dev borra el bodyB en Play Mode sin pasar por `DeleteEntityCommand`, el constraint queda huérfano (Jolt lo invalida internamente; no crashea pero queda allocado hasta el siguiente `dirty = true`).
- **Limits para Distance/Point** — Distance acepta min/max diferentes (cuerda flexible), pero el editor v1 solo expone DragFloat — no hay validación de `max >= min`. Si el dev mete max < min, Jolt clamea sin error visible. Agendable polish.
