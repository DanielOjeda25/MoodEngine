## Qué entregó

### Bloque A — Schema MaterialAsset extendido

`enum class BlendMode : u8 { Opaque = 0, Translucent = 1, Additive = 2 }` en `MaterialAsset.h`. Campos nuevos:

- `BlendMode blendMode = BlendMode::Opaque`
- `f32 opacity = 1.0f` (0-1)
- `f32 ior = 1.0f` (1.0 sin refracción, 1.33 agua, 1.5 vidrio, 2.42 diamante)
- `f32 refractionStrength = 0.0f` (0-1, multiplicador del offset screen-space)

Serialización aditiva en `AssetManager_Material.cpp`: keys `blend_mode` (`"opaque"`/`"translucent"`/`"additive"`), `opacity`, `ior`, `refraction_strength`. Los `.moodmat` viejos cargan con `Opaque` + defaults. **Solo se persisten los campos cuando `blendMode != Opaque`** para no contaminar diffs históricos.

### Bloque B — Shader emite alpha + refracción screen-space

`pbr.frag` + `pbr_graph_template.frag` ganan uniforms:

```glsl
uniform int       uBlendMode;
uniform float     uOpacity, uIor, uRefractionStrength;
uniform sampler2D uBackbufferCopy;  // unit 7
uniform vec2      uScreenSize;
```

Branching en el final de `main`:

- **Opaque** (`uBlendMode==0`): `FragColor = vec4(lighting, 1.0)` y `return` — zero overhead.
- **Fresnel-driven auto-opacity** (integrado, no opt-in): `fresnel = pow(1 - clamp(NdotV, 0, 1), 5)` + `effectiveOpacity = mix(uOpacity, 1, fresnel * 0.5)` — bordes más opacos, frente más transparente.
- **Additive** (`uBlendMode==2`): `FragColor = vec4(lighting * effectiveOpacity, effectiveOpacity)`.
- **Translucent**: sample del `uBackbufferCopy` con offset `nViewXY * (uIor - 1) * uRefractionStrength * 0.1` → `mix(refracted, lighting, effectiveOpacity)`.

Bug fix mid-implementación: tenía referencia a `m_viewportSize` (no existe). Cambio a `glm::vec2(static_cast<f32>(fbW), static_cast<f32>(fbH))` usando los miembros del FB.

### Bloque C — Backbuffer copy para refracción

`SceneRenderer::blitSceneToBackbufferCopy()` captura el FBO activo via `glGetIntegerv(GL_DRAW_FRAMEBUFFER_BINDING, ...)`, blittea con `glBlitFramebuffer` (color attachment, formato preservado HDR `RGBA16F`) a `m_backbufferCopyFb`. Lazy alloc + resize cuando cambia el viewport. Se llama 1 vez por frame, justo antes del translucent pass.

### Bloque D — Translucent pass en SceneRenderer

Nuevo `TranslucentDraw { Entity entity; glm::vec3 worldCenter }` + bucket `std::vector<TranslucentDraw> translucents` en `BatchingResult`. `RenderBatching::groupByBatch` ramifica antes de `instanced`/`nonBatchable`: si `material->blendMode != Opaque`, va a `translucents` con `worldCenter` precomputado.

`PBR::translucentPass`:

1. Sort back-to-front por `glm::dot(worldCenter - camera, viewForward)` desc. Tiebreaker estable por `static_cast<u32>(entity.handle())`.
2. Blit del backbuffer (delega a Bloque C).
3. Bind `m_backbufferCopyFb->colorTexture()` a unit 7.
4. Per-entity: `glBlendFunc` según mode (`SRC_ALPHA/ONE_MINUS_SRC_ALPHA` para Translucent, `SRC_ALPHA/ONE` para Additive), `glDepthMask(GL_FALSE)`, depth-test ON, set uniforms, draw.
5. Restore GL state.

Llamado en `SceneRenderer_Render.cpp` después de pase opaco + skybox + brushes, **antes** de SSAO/SSR/bloom/particles. Razón: los translúcidos leen el color de los opacos; los screen-space passes posteriores no afectan al backbuffer copy.

### Bloque E — UI Inspector Blending

`InspectorPanel_MeshRenderer.cpp` gana `CollapsingHeader` "Blending" con:

- `ImGui::Combo` BlendMode (Opaque/Translucent/Additive).
- `ImGui::SliderFloat` Opacity (0-1), disabled si Opaque.
- `ImGui::SliderFloat` IOR (1.0-2.5), disabled si Opaque o Additive. Tooltip con presets.
- `ImGui::SliderFloat` Distorsión (refractionStrength, 0-1), disabled si Opaque.

Undo via `pushEditIfDone<u32>` (cast BlendMode→u32 — el `InspectorEditTracker` variant no incluye `int`). 4 keys i18n nuevas (es + en).

### Bloque F — Shader Graph OutputPBR 6º input Opacity

`OutputPBR` pasa de 5 → 6 inputs (Albedo, Metallic, Roughness, Normal, Emissive, **Opacity**) en `ShaderGraphAsset.cpp`. Marker `__SHADERGRAPH_OPACITY__` agregado a `requiredMarkers` en `ShaderGraphGenerator.cpp`. Back-compat: assets `.moodshader` con 5 inputs caen al literal `1.0`. Test regression confirma que un OutputPBR legacy compila sin error.

En el template: `float graphOpacity = __SHADERGRAPH_OPACITY__; float baseOpacity = clamp(graphOpacity * uOpacity, 0.0, 1.0);` — el slider del Inspector funciona como "master volume" multiplicando lo que sale del grafo.

## Bug cazado durante validación visual + fix

**Síntoma:** *"EL DE DISTORSION FUNCIONA, esta viendo los logs? porque estoy poniendo cada sample, y aunque los veo transparentes, los veo todo iguales"*.

**Causa:** los samples originales de glass + hologram usaban `Fresnel(power=N)` conectado al input Opacity del OutputPBR. Resultado: face-on (dot(N,V) ≈ 1), `Fresnel = pow(1 - 1, N) = 0`, opacity=0 → el cubo se volvía invisible face-on y solo se veía el fondo refractado. Los tres samples se veían "iguales" porque face-on todos mostraban solamente el fondo.

**Fix:** rewrite de glass y hologram con opacity literal (no Fresnel-driven). Hologram también pierde el `Lerp(black, cyan, Fresnel)` y va a HDR cyan direct al emissive — así "glowea" en todas las direcciones, no solo en los bordes. Cada sample ahora tiene una identidad visual clara face-on.

## Decisiones clave (detalle en `DECISIONS.md`)

1. **Split en F2H63 (base + refracción) + F2H64 (OIT + translucent shadows).** Razón: 30h en un solo hito es riesgo de regresión + el dev pidió validar la base antes de mover a correctness multi-vidrio.
2. **Screen-space distortion sobre full Snell refraction.** Convención Unity URP / Unreal "Distortion". Resultado visual indistinguible en geometría plana / convexa simple; ~zero costo extra.
3. **Fresnel-driven auto-opacity integrado, no opt-in.** Look "vidrio real" sin que el dev tenga que armar el grafo Fresnel cada vez.
4. **Translucent pass antes de SSAO/SSR/bloom.** Los translúcidos leen el color de los opacos; los screen-space passes posteriores no afectan al backbuffer copy.
5. **Backbuffer copy 1x por frame, no per-translucent.** Sort back-to-front sobre snapshot estable; suficiente para escenas <50 vidrios. OIT en F2H64 si emerge demanda.
6. **Persistir blending solo si `blendMode != Opaque`.** Back-compat aditivo + diffs limpios en `.moodmat` históricos.
7. **OutputPBR 6º input + fallback literal `1.0`.** Back-compat con shaders viejos sin tener que migrar assets.

## Limitaciones conocidas v1 (agendadas a F2H64)

- **OIT** — sort back-to-front puede flickerear con muchos translúcidos solapados o follaje denso.
- **Sombras translúcidas** — los vidrios proyectan sombra opaca (o ninguna). Sombras tintadas requieren extender el shadow atlas.
- **Refracción full Snell** — screen-space hack es indistinguible para geometría simple; rayo a través del material si emerge demanda.
- **Translucent en SSAO/SSR** — leen el depth attachment del pase opaco. Aceptable v1 (convención estándar Unity URP / Unreal).
- **Multiply blend mode** — sin demanda específica.
- **Refracción screen-space en bordes de pantalla** — coord desplazada puede caer fuera del backbuffer copy. Mitigado con `clamp(uv, 0.001, 0.999)`; mejora futura: fade en bordes.
