## Contenido original (backfill AUDIT-2 mecanico)

Migrado tal cual desde HITOS.md durante AUDIT-2 (2026-05-17). No reformateado — preserva el wall-of-text original para no perder informacion. Reformateo a estructura limpia agendado a un audit futuro si emerge necesidad de leerlo.

Bloques A-G, **pivot temporal de Sub-fase 2.6 a UX del editor** entre F2H58 (color grading) y F2H6X (siguiente render polish). **Origen**: post-cierre F2H58 el dev detectó durante el tour visual que usaba un Box brush aplastado como suelo y pidió la primitiva Plano + primitivas clásicas adicionales para no modelar todo desde cero. La conversación escaló a UX reorg general: sacar primitivas del menú top-level + del Toolbar lateral, agruparlas en el modal "+ Crear Entidad" del panel Escena (workflow SFM-style ya consolidado en F2H57), y migrar las herramientas (Mover/Rotar/Escala/Cara) a un overlay flotante sobre el viewport (estilo Blender). **Bloque A — primitiva Plano** (handler `handleAddPlaneBrush` en `EditorProjectActions_Brush.cpp`): preset de Box CSG con dimensiones 10m × 0.05m × 10m optimizadas para suelo. Convención Unity Plane / Unreal Plane / Godot PlaneMesh. Internamente sigue siendo un Box CSG estándar — el dev lo edita con el mismo gizmo y se serializa al `.moodmap` igual. Entry "Plano (suelo)" arriba de Box en el menú Brush > Añadir (pre-Bloque E). **Bloque B — Cono + Cápsula + Quad**: 3 primitivas adicionales. **Cono**: nueva función CSG `Csg::makeConeBrush(transform, segments=16)` con N planos laterales triangulares convergiendo al apex `(0, +0.5, 0)` + 1 cap base. Geometría calculada: `normal = (cos(θ)*cos(α), sin(α), sin(θ)*cos(α))` donde `α = atan(0.5/1) = atan(0.5)`, `lateralDist = -sin(α) * 0.5`. **Quad**: preset Box 1×0.05×1 (versión chica del Plano para sprites/billboards). **Cápsula**: preset Sphere dodecaédrica estirada en Y 2× (no es cápsula técnica con paredes laterales rectas, sino elipsoide alargado — cubre el caso común "personaje proxy / pildora / pilar redondeado"). **Toro**: SKIPPED en v1 — un toro no es geométricamente convexo (tiene un agujero) y el sistema CSG de MoodEngine asume brushes convexos. Workaround documentado: 2 cilindros + boolean Subtract manual; o esperar F2H60+ que evalúa modificadores Blender-style. **Bloque D — limpieza del Toolbar lateral**: removidos los botones "Box" y "Cylinder" del panel Toolbar (eran spawn de geometría, no herramienta de edición). El Toolbar queda solo con herramientas de edición (Mover/Rotar/Escala/Cara). Las primitivas tienen un único punto de entrada (el modal). **Bloque E — primitivas al modal "+ Crear Entidad"** (`EditorProjectActions_CreateEntity.cpp`): refactor del modal `renderPickFromLoadedMeshesModal` con TabBar de 2 sub-secciones: "Meshes del proyecto" (flujo SFM original) + "Primitivas" (grid 3-col de las 11 brushes). Cada botón dispara el `ProjectAction` correspondiente y cierra el modal. El menú top-level "Brush > Añadir" eliminado completamente — el panel Brush queda solo con "Operaciones Booleanas". Punto único de "como creo geometría". **Bloque F — toolbar flotante overlay sobre el viewport** (`ViewportPanel.cpp::drawViewportToolsOverlay`): sub-window ImGui posicionada al top-left de la imagen del viewport con `ImGui::SetNextWindowPos(imageMin + (8,8), ImGuiCond_Always)`. Flags `NoDecoration | NoMove | NoDocking | AlwaysAutoResize | NoSavedSettings | NoFocusOnAppearing | NoNav | NoBackground` + `WindowBorderSize=0` + `WindowBgAlpha=0` para que solo se vean los botones (estilo Blender). 4 botones icon-only: Mover (`ICON_FA_ARROWS_UP_DOWN_LEFT_RIGHT`), Rotar (`ICON_FA_ROTATE`), Escala (`ICON_FA_UP_RIGHT_AND_DOWN_LEFT_FROM_CENTER`), Face (texto "F" — pedido explícito del dev "deberia ser F de faces"). El panel Toolbar original queda como dead code linkeable (no en `m_panels`); el ViewportPanel recibe `setEditorUi(this)` para conectar los buttons al `requestGizmoMode` / `requestToggleFaceMode`. **Bug pre-existente del MenuBar fixeado en F2H58 Bloque F**: no aplica a F2H59 (ya estaba). **Followup UX polish del modal Crear Entidad**: (a) título ImGui del popup era literal `pick_mesh_modal` — fixeado a "Crear entidad" usando sufijo `##pick_mesh_modal` para que ImGui muestre el label traducido pero matchee el OpenPopup. (b) Botón "Cerrar" inferior eliminado — se cierra con la X arriba (estandard ventana, `BeginPopupModal(&active)` pinta la X). (c) Footer del modal con labels engine-y: "Importar..." (file picker SO) + "**Empty**" (placeholder vacío) — termino "Empty" pedido explícito del dev tras yo proponer "Nuevo (vacío)". (d) Altura uniforme del contenido de las tabs con `BeginChild` 320px para que el footer quede en la misma posición vertical sin importar qué tab esté activa (pedido del dev: *"en primitivas no esta el footer bien posicionado"*). (e) Botón "F" del overlay con ID único `##envreset_xxx` — wait, este punto era de F2H58, no aplica aquí. Los 4 botones del overlay tienen labels visibles únicos así que no necesitan ID suffix. **Decisiones**: (1) **Toro skipped v1** — no convexo, requiere CSG no-soportado o N segmentos auto-spawned. (2) **Cápsula = Sphere estirada Y**, no cápsula técnica — trade-off de simplicidad; si emerge demanda de paredes laterales rectas, follow-up con `makeCapsuleBrush()` real (cilindro + 2 hemisferios). (3) **Modal "+ Crear Entidad" como punto único** — quita las primitivas del menú Brush > Añadir y del Toolbar. Workflow SFM/Hammer consistente: 1 click en panel Escena → modal → click en primitiva. Menos overhead UI. (4) **Toolbar lateral fuera del dockspace** — las herramientas viven como overlay flotante sobre el viewport (estilo Blender / Unity / Unreal moderno / Godot 4). Hammer 2004 usa toolbar lateral fijo; los engines modernos usan overlay flotante. MoodEngine sigue convención moderna. (5) **Background del overlay totalmente transparente** (alpha 0 + border 0 + NoBackground flag) — pedido del dev: *"le podemos dejar el fondo transparente para que solo sean botones flotantes?"*. (6) **"F" como label del botón Face** sobre `ICON_FA_VECTOR_SQUARE` — pedido del dev. Homogeneización del sistema de iconos del editor agendada como follow-up. (7) **Footer del modal con labels engine-y** ("Importar..." + "Empty") sobre los nombres verbosos previos ("Importar desde archivo..." / "Crear vacía (placeholder)") — término "Empty" del vocabulario universal de engines (Unity Empty GameObject / Unreal Empty Actor / Godot Empty Node). **Limitación arquitectónica destacada durante el tour** (NO scope F2H59, agendada a F2H60): **los brushes con `RigidBodyComponent` Dynamic NO caen visualmente** — el mesh cache del brush se construye con `worldMatrix` incrustado en los vértices y solo se rebuilda si `bc.dirty` es true. El body de física SÍ se simula correctamente (Jolt lo cae), pero el render del brush queda desactualizado porque `t.position` cambia y el cache no se refresca. Esto es consistente con la convención Source/Hammer 2004: brushes son geometría estática del mapa, props (meshes) son los objetos con física dinámica. **F2H60** adopta el Source paradigm explícitamente: rename "Brush" → "Estructura" en UI; sub-secciones "Estructura" + "Objetos" en el modal Crear Entidad; meshes procedurales generados en runtime para Cubo/Esfera/Cilindro/Cápsula/Cono que sí soportan física dinámica; warning en Inspector si el dev pone RigidBody Dynamic sobre un Brush. **Bug detectado durante el primer tour del overlay** (fixeado): los botones reset del Environment panel tenían IDs ImGui colisionando — wait, ese fue F2H58. Aquí el bug fue el carácter Unicode `…` U+2026 que renderizaba como `?????` en la font default (pre-F2H58 cierre) — pero ese ya estaba fixeado. Lo que sí fue de F2H59: el overlay flotante mostraba líneas finas del frame de la sub-window aunque el background era transparente; fix con `PushStyleVar(WindowBorderSize, 0)` + flag `NoBackground` explícito. Pedido del dev: *"se sigue viendo unas lineas finas"*. **Pregunta del dev resuelta durante el tour** (Hammer reference): cómo Hammer Editor 2004 organizó las herramientas — toolbar lateral fijo vertical con TODAS las tools en columna, NO overlay flotante. MoodEngine ya tomó el approach moderno (Blender-style) para el viewport principal. Para el workspace `map_editor` se mantiene el panel **Map Tools** existente (más cercano a Hammer) — F2H60 candidate evalúa híbrido: Map Tools dockable + opcionalmente desplegable como overlay del viewport ortográfico. **Suite 926/9705 verde** (sin tests nuevos: primitivas nuevas son glue UI sobre código CSG/render existente; el algoritmo del Cono se valida visualmente).
