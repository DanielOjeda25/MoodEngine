Bloques A-F (CSM core) + iter 1-5 polish, **reset al plan original Fase 2** tras descartar pivot al Source paradigm propuesto post-F2H59. **Origen**: el plan original Sub-fase 2.6 contemplaba CSM como sucesor de Hito 16 (single shadow map estático). El agente propuso reorientar F2H60 al Source paradigm (brushes = estructura estática + meshes procedurales = objetos físicos) pero el dev pushback: *"no quiero alejarme tanto de la vision y el plan original"* — reset a CSM clásico, follow-ups del Source paradigm + Map Tools híbrido + modificadores Blender + brushes con física dinámica + overlay context-aware acumulados en `docs/BACKLOG.md` (15+ items). **Bloque A — `OpenGLShadowMapArray`**: wrapper sobre `GL_TEXTURE_2D_ARRAY` 2048×2048×N + 1 FBO con `glFramebufferTextureLayer` para renderear cada cascada como layer separado. API mínima: `bindLayerAsTarget(idx)`, `bindAsSampler(unit)`, `size()`, `cascadeCount()`. **Bloque B — `ShadowMath.h` extensión**: `computeCsmSplits(near, far, count, lambda)` (PSSM Zhang 2006, blend lineal/log con lambda 0-1), `computeCascadeShadowMatrices(view, proj, splitNear, splitFar, lightDir, smSize)` (bounding sphere del sub-frustum + texel snapping para evitar shimmering). `kMaxCsmCascades = 4`. **Bloque C — refactor `ShadowPass`**: API `recordCsm(scene, assets, renderer, lightDir, cameraView, cameraProj, cascadeCount, lambda)` reemplaza el record single-map. Infiere `cameraNear/Far` de la `proj` matrix; itera 3 tipos de casters (MeshRenderer / BrushComponent / CompiledMeshComponent — ver iter3 polish). `lightSpaceMatrix(i)` + `cascadeSplit(i)` exponen los datos al lit shader. **Bloque D — shaders `pbr.frag` + `pbr.vert` (+ variants `lit`/`pbr_instanced`/`pbr_skinned`)**: `sampler2DShadow` → `sampler2DArrayShadow`, uniforms nuevos `uCascadeCount`, `uLightSpaces[4]`, `uCascadeSplits[5]`. Función `sampleShadowCascade(worldPos, idx)` con hardware PCF 4 taps + 3×3 manual (36 muestras efectivas), bias escalado por `(cascadeIdx + 1)`. Función `sampleShadow(worldPos)` selecciona cascada por `vViewSpaceZ` con blend lineal en el último 15% del rango (evita banding visible al cruzar fronteras). Vert pasa `vViewSpaceZ = abs((uView * worldPos).z)` y removida la pre-F2H60 `uLightSpace` / `vLightSpacePos`. **Bloque E — wireup `SceneRenderer`**: cache F2H42 (bounding-sphere hash) eliminado — CSM depende de la matriz de cámara que cambia casi siempre. Si `cascadeCount` cambia via Inspector, `m_shadowPass->recreate()`. **Bloque F — `EnvironmentComponent` extendido** + Inspector + i18n. **Suite tests**: 7 test cases nuevos en `tests/test_shadow_proj.cpp` (split scheme 1 cascada, 4 cascadas lambda 1.0/0.0/hybrid, cascadeCount=0 sanity, sub-frustum sizes ordenados, centro AABB dentro de ortho). Tests roundtrip Environment ✓ post-cleanup del campo `csmEnabled`. **5 iteraciones de polish UX surgidas durante el tour del dev (todas validadas)**: **iter1 — defaults OFF** (`bloomEnabled = false`, `ssaoEnabled = false`, `csmEnabled = false` inicialmente) por pedido explícito: *"todo deberia estar desactivado por defecto y solo activar a gusto"*; alineado en `Components.h`, `SceneSerializer.h`, `EntitySerializer.cpp` write conditions + read defaults, `SceneRenderer.cpp`. **iter1 — botones reset con nombre de sección** (`drawSectionResetButton(idSuffix, sectionLabelKey, applyFn)` — label visible `⟲ Restablecer <Sección>`) por pedido: *"los botones de restablecer confunden, no realmente muestran bien a que seccion pertenecen"*. **iter1 — GL warnings "texture object 0 ... shadow sampler"**: shadow map ahora se bindea SIEMPRE (no condicional a `shadowEnabled`) porque GL valida samplers de la draw call aunque el shader no los samplee en el branch dinámico (`uShadowEnabled==0` fast path). **iter1 — Color Grading dropdown `?????`**: bug clásico C++ lifetime — `I18n::T(...).c_str()` retornaba puntero a `std::string` temporal liberado; fix asignar a `std::string previewStr` local antes de `.c_str()`. **iter2 — separación visual de secciones** + revert labels a "Restablecer" plano por pedido: *"solo diga Restablecer, pero no estan bien separados cada seccion"*; helper `drawSectionDivider()` (Dummy(4)+Separator()+Dummy(4)) entre cada CollapsingHeader, label de botón sin sufijo. **iter2 — `csmEnabled` master switch eliminado**: el dev activaba CastShadows en la luz directional pero las sombras no aparecían porque también requería `Environment.csmEnabled = true`. Gate doble = confusión. Solución: eliminar `m_csmEnabled` global; el gate de sombras es **solo per-light** (`LightComponent::castShadows`). Limpieza profunda en `Components.h`, `SavedEnvironment`, `SceneRenderer.{h,cpp}`, `SceneRenderer_Render.cpp:93`, i18n keys (`csm_enabled` reemplazada por hint `csm_hint`). **iter2 — tab "Luces" en modal Crear Entidad**: 2 nuevas `ProjectAction::AddDirectionalLight` / `AddPointLight` + handlers `handleAddDirectionalLight` (spawn (0,3,0), dir=(-0.3,-1,-0.2), castShadows=true) / `handleAddPointLight` (spawn (0,2,0), radius=10m, color cálido); modal `renderPickFromLoadedMeshesModal` con 3 tabs (Meshes / Primitivas / Luces). Pedido del dev: *"deberia poder crearse luces desde el panel de entidades"*. **iter3 — bug shadow casters faltantes**: el `ShadowPass::recordCsm` solo iteraba `MeshRendererComponent`, así que los brushes CSG (primitivas spawneadas) **no proyectaban sombras**. Fix: añadir paths para `BrushComponent` (usando `bc.meshCache` poblado lazy por el PBR pass, primer frame puede estar vacío) y `CompiledMeshComponent` (MoodPlayer). Log diagnóstico `ShadowPass casters: total=N (MeshRenderer=X, Brush=Y, CompiledMesh=Z), brushes_sin_cache=W`. **iter3 — bug `uModel` brush en shadow pass**: las posiciones de `BrushMeshVertex` viven en **LOCAL space** (`v.position = p` en `BrushMesh.cpp:154` sin multiplicar por worldMatrix); el worldMatrix solo se usa para UVs lockToWorld. Mi fix iter3 inicial usaba `uModel = identity` → todos los brushes se renderaban al origen del shadow map → la sombra del cubo quedaba clavada en (0,0,0) y al moverlo se "desconectaba". Fix correcto: `uModel = t.worldMatrix()` (mismo criterio que el PBR pass). **iter4 — `applyEnvironmentFromScene` movido ANTES del shadow pass**: pre-fix los sliders de cascadas/lambda tenían 1 frame de delay y "no respondían en tiempo real". **iter4 — `uShadowBias` 0.005 → 0.0015**: el bias viejo era ~0.25m de "lift" en NDC depth → peter-panning brutal al apoyar un brush sobre el piso (la sombra "desaparecía" porque el bias hacía que el receiver pareciera estar más cerca de la luz que el caster). **iter5 — bug crítico fórmula extracción near/far del proj matrix**: las fórmulas estaban INVERTIDAS para GLM perspective right-handed. Pre-fix log mostraba `near=100, far=0.1, splits=[100.00 | 100.25 | ... | 101.00]` → cascadas proyectaban fuera de cuadro → mover lambda/cascadas no surtía efecto visible (cambiaba el cálculo pero el resultado seguía en rango basura). Fórmula correcta: `near = m32 / (m22 - 1)`, `far = m32 / (m22 + 1)`. **iter5 — logs diagnósticos**: 3 puntos (applyEnvironmentFromScene LEE Environment.CSM, ShadowPass params con splits computados, ShadowPass casters por tipo) — solo logean al cambiar valores (sin spam por frame). **Decisiones**: (1) **CSM clásico sobre Source paradigm** — reset al plan original. Backlog de follow-ups Source paradigm + Map Tools híbrido + modifiers Blender + brushes con física dinámica documentado en `docs/BACKLOG.md` (no en el plan F2H60). (2) **Gate de sombras solo per-light** (eliminado `csmEnabled` global) — `LightComponent::castShadows` es la fuente única de verdad; el panel Environment solo tiene knobs de calidad (cascadas + lambda). (3) **`uModel = worldMatrix` para brushes en shadow pass** — mismo criterio que el PBR pass; las vertices del brush son LOCAL space (worldMatrix solo se usa para UVs lockToWorld). Bug histórico de la primera implementación. (4) **`applyEnvironmentFromScene` ANTES del shadow pass** — para que los sliders surtan efecto en el mismo frame. No tiene side-effects de GL state (solo poblar miembros), así que es seguro mover. (5) **Log diagnóstico per-cambio + diagnóstico de orden de magnitudes** — al detectar `near=100, far=0.1` los logs hicieron triviales identificar el bug iter5. Conservados como guía de debug para futuras sesiones (low-spam: solo cambian frame-a-frame). (6) **PSSM lambda hybrid 0.5 como sweet spot** — lambda=0 puro lineal subutiliza cascadas cerca; lambda=1 puro log subutiliza cascadas lejos; 0.5 balance práctico (Zhang 2006). (7) **Bias 0.0015 + escala por cascada (×1,×2,×3,×4)** — equilibrio entre evitar shadow acne (bias chico) y evitar peter-panning (bias grande); el escalado compensa que cascadas lejanas tienen texeles más grandes. (8) **Sin shadow para luces puntuales en v1** — cube map shadows + tetraedrón shadows agendados como follow-up (Hito 22+ del plan original aún más adelante). **Suite tests 933/9712 verde** (7 nuevos en `test_shadow_proj.cpp`).
