Bloques A-F, **= F2H20 del plan original Fase 2 (Sub-fase 2.3 — Renderer)**. Continúa el orden del plan original tras F2H60 (CSM): F2H17 ✅ material editor lite, F2H19/F2H56 ✅ SSAO, F2H21/F2H55 ✅ bloom, F2H22/F2H60 ✅ CSM, **F2H20/F2H61 → SSR**, queda F2H18/F2H62 → shader graph runtime para cerrar Sub-fase 2.3. **Origen**: retoma plan original tras consulta del dev: *"revisa el plan y si SSR es el siguiente, vamos"*. **Filosofía**: polish no-destructivo + engine-grade + industry-standard. Algoritmo Morgan McGuire & Mike Mara "Efficient GPU Screen-Space Ray Tracing" (2014), implementación linear DDA en view-space (sin Hi-Z optimization — suficiente para v1, diferido si emerge presión de perf). **Bloque A — G-buffer parcial via MRT**: extensión de `OpenGLFramebuffer` con parámetro `withNormalRT = false` al ctor. Cuando está y el format es `HDR`, crea `m_normalTexture` (RGBA16F) attacheada a `COLOR_ATTACHMENT1` + `glDrawBuffers(2, ...)`. Accessor `glNormalTextureId()`. El `m_sceneFb` del `SceneRenderer` ahora se construye con `withNormalRT=true`. Los 3 PBR vertex shaders (`pbr.vert`, `pbr_instanced.vert`, `pbr_skinned.vert`) emiten varying `vViewSpaceNormal = normalize(mat3(uView) * vWorldNormal)`. El `pbr.frag` declara `layout(location=1) out vec4 NormalRT` y escribe `vec4(normalize(vViewSpaceNormal), 1.0)`. El alpha `1.0` marca "pixel escrito por shader PBR" — pixels no-PBR (skybox/particles/debug) no escriben location 1 y quedan en `(0,0,0,0)` gracias al clear adicional `glClearBufferfv(GL_COLOR, 1, zero)` post-clear global. SSR descarta pixels con `alpha < 0.5`. **Bloque B — `shaders/ssr.frag`**: ray marching linear DDA en view-space. Lee 3 texturas: scene color, depth attachment, normal RT. Reconstruye `viewPos` desde `(uv, depth)` via `uInvProj`. `V = -normalize(viewPos)`, `R = reflect(-V, N)`. Marcha N pasos con `stepSize` constante. Hit cuando `gap > 0 && gap < uThickness` → sample del scene color en hitUv → `FragColor = baseColor + reflection * uIntensity`. Sin hit → `baseColor` inalterado (fallback al cubemap IBL ya bakeado en sceneColor). Reusa `ssao.vert` como fullscreen triangle. **Bloque C — `SSRPass.{h,cpp}`**: patrón establecido (single draw call, sin FBs internos, save/restore GL state). Firma `apply(srcColor, srcGbuffer, dst, proj, invProj, maxSteps, thickness, stepSize, intensity) → bool` con 2 FBs source: srcColor (puede venir post-SSAO con AO aplicado) + srcGbuffer (`m_sceneFb` original con depth+normal). Skip si `intensity <= 0` o gbuffer sin normal RT. **Bloque D — wireup en `SceneRenderer::endFrame`**: nuevos members `m_ssrPass` + `m_ssrFb` (HDR sin normal RT propio) + try/catch en setup. Pipeline post-F2H61: `m_sceneFb → SSAO → afterSsao → SSR (color del afterSsao, gbuffer del sceneFb) → afterSsr → Bloom → ColorGrading → PostProcess`. SSR **después** de SSAO para que el color base ya tenga AO + bloom/color grading downstream apliquen al reflejo también. Cero regresión si SSR off (default) o gbuffer ausente. **Bloque E — persistencia + UI**: `EnvironmentComponent` + `SavedEnvironment` extendidos con 5 campos (`ssrEnabled` default OFF, `ssrMaxSteps` u32 default 32, `ssrThickness` default 0.5, `ssrStepSize` default 0.2, `ssrIntensity` default 0.5). Serialización aditiva mismo patrón que bloom/SSAO/CG/CSM. Sección "Reflejos screen-space (SSR)" en `InspectorPanel_Environment.cpp` bajo el wrapper Post-Process. 4 sliders (intensidad / pasos / tamano paso / espesor) + checkbox + reset button + hint disabled. 7 i18n keys nuevas (es + en). Undo via `pushEditIfDone`. `ssrMaxSteps` como `u32` (no `i32`) para encajar en el `InspectorEditTracker` variant existente. **Bloque F — validación visual + cierre**. **Decisiones**: (1) **MRT G-buffer vs depth-prepass separado** — MRT extiende el sceneFb sin duplicar draw calls. Cost: ~8MB extra a 1080p. Trade-off de memoria por simplicidad. (2) **Linear DDA en view-space** sobre perspective-correct o Hi-Z — McGuire 2014 simple sin importance sampling. Suficiente para v1. Hi-Z diferido si emerge presión perf. (3) **SSR después de SSAO** — el reflejo recibe AO de la superficie (físicamente: reflejo en esquina ocluida debería ser más oscuro). Trade-off: el reflejo en sí NO recibe AO de la zona donde apunta. Visualmente casi imperceptible. (4) **Sin per-material toggle en v1** — SSR aplica a TODOS los pixels con alpha > 0.5 del normal RT con la misma intensity global. Per-material agendado al BACKLOG. (5) **`ssrMaxSteps` como `u32`** — variant del InspectorEditTracker no incluye i32. Mas simple casteear que extender el variant. (6) **Default OFF** — iter1 F2H60 polish: efectos visuales opt-in. (7) **Fallback al cubemap IBL implícito** — sin hit, retorna baseColor (cubemap spec ya bakeado por el PBR pass). **Limitaciones conocidas v1**: reflejos cortados al borde de pantalla (sin fade-out), sin temporal accumulation/TAA, sin per-material toggle, sin refracciones, sin Hi-Z. **Validación visual del dev** (2026-05-17): cubo missing-texture sobre piso PBR → reflejo visible del cubo en el piso. Cita verbatim: *"se ve decente"*. **Feedback UX detectado durante el tour (NO scope F2H61, agendado al BACKLOG ítem 1.0)**: el dev tuvo que crear un cubo placeholder solo para hostear el `EnvironmentComponent` — *"el cubo con la textura missing es mi environmentCOmponent, mas adelante me gustaria que tenga un icono y una opcion propia, porque de la forma que hago ahora es agregar un empty y ahi agregar el componentEnviroment y no le veo el sentido si es un componente global"*. Convención Unity Volume / Unreal PPV / Godot WorldEnvironment — entrada dedicada en el modal Crear Entidad + icono propio. **Suite tests verde** (sin tests nuevos: SSR es shader visual no headless-testeable; serialización aditiva cubierta por el patrón existente).
