## Contenido original (backfill AUDIT-2 mecanico)

Migrado tal cual desde HITOS.md durante AUDIT-2 (2026-05-17). No reformateado — preserva el wall-of-text original para no perder informacion. Reformateo a estructura limpia agendado a un audit futuro si emerge necesidad de leerlo.

Bloques A-G + followup, **pivot temporal de Sub-fase 2.6 a UX del editor** entre F2H56 (SSAO) y F2H58 (color grading). **Origen**: durante el tour visual de F2H56 el dev detectó 3 problemas UX — (1) no había "Crear Entidad" button visible, obligando a poblar la escena vía Demos del menú Ayuda (muleta), (2) la vista ortográfica SIDE (ZY) dibujaba brushes con eje invertido, (3) los Demos eran muleta una vez existiera workflow real. F2H57 cierra los 3 + un workflow estilo Hammer Editor / Source Film Maker para crear y mutar entidades. **Bloque A — fix SIDE ortho**: `OrthoCamera::rightAxis()` retornaba `{0, 0, +1}` para `OrthoView::SideZY` pero `glm::lookAt(eye=+X, target=-X, up=+Y)` produce camera right = `cross(forward, up) = cross(-X, +Y) = (0, 0, -1)` — sign mismatch hacía que el draw apuntara invertido respecto al cursor. Fix: flip sign a `{0, 0, -1}`. Pan unaffected (drag * -scale algorithm robusto a sign). Dev validó: *"el bug side esta corregido"*. **Bloque B — "+ Crear Entidad" button + workflow inicial**: botón nuevo arriba de la lista del HierarchyPanel (`+ Crear Entidad`, i18n `editor.panel.hierarchy.create_entity`). Primera versión abría file picker del SO directo. **Bloque C — context menu right-click sobre entidad**: `ImGui::BeginPopupContextItem` por entidad con "Cambiar tipo de entidad..." + "Eliminar". Mismo handler de delete reusado desde el toolbar shortcut F2H22. `EditorUI` extiende con `requestEntityConvertModal(entt::entity)` + `requestDeleteSelectedEntity()` flags. **Bloque D — modal "Convertir entidad"** (`renderConvertEntityModal()` en `EditorProjectActions_CreateEntity.cpp`): 4 kits v1 — NPC con diálogo (`TriggerComponent` + `DialogComponent` con path placeholder), Item pickeable (`TriggerComponent` + `ItemPickupComponent` con path placeholder), Luz puntual (`LightComponent` type Point), Luz direccional (`LightComponent` type Directional con castShadows=true). Cada kit es **aditivo no-destructivo** — agrega componentes sin remover los existentes. Botones disabled si el componente ya está aplicado. Modal NO se cierra al aplicar (permite stack de kits en la misma entidad). **Limitación v1 documentada**: el paso convert NO es undoable (snapshot pre/post de componentes diferido a v2). Edits individuales post-convert SÍ son undoable via `InspectorEditCommand` F2H32. **Bloque E — eliminar Demos del menú Ayuda**: removidas las entries del menú `Ayuda > Demos` en `MenuBar.cpp` (~80 líneas). Approach minimal-risk: solo se borran menu entries + handlers; `DemoSpawners_*.cpp` quedan como dead code para no romper helpers compartidos (`ensureDemoIntroDialogExists` etc.). Si emerge demanda, hardening completo del cleanup en hito futuro. **Bloque F — validación visual + followup en mecánicas**: dev validó el workflow base. Pidió 3 ajustes — (a) opción de elegir mesh ya importado en el proyecto sin re-navegar SO (convención Source Film Maker / asset browser de Unity), (b) limpieza del Welcome modal (F2H44 Fox demo + F2H50 narrative demo buttons sin sentido sin workflow externo), (c) fix de centrado del Welcome en pantallas grandes (>720p). **Bloque F followup — SFM-style mesh picker**: `renderPickFromLoadedMeshesModal()` en `EditorProjectActions_CreateEntity.cpp` itera `AssetManager::meshCount()` y `meshPathOf(id)` listando todos los meshes ya cargados (skip primitivos `__missing_cube`/`__primitive_sphere` por prefix `__`). Click en un item spawnea entidad con ese mesh (reusa lógica de spawn con AABB Y bottom-anchor + auto-escala). Final UX: click en `+ Crear Entidad` → abre directo el modal SFM-style con la lista de meshes del proyecto (sin popup intermedio); modal incorpora dos botones de acción al pie — "Importar desde archivo..." (file picker del SO) y "Crear vacía (placeholder)" — y un "Cerrar". **Followup welcome modal**: F2H44/F2H50 demo buttons removidos. Bug centrado fixeado en `EditorUI::drawWelcomeModal` — usaba `ImGuiCond_Appearing` y el viewport del primer frame todavía no tenía su tamaño final en pantallas grandes; cambiado a `ImGuiCond_Always` con `WorkPos + WorkSize/2` para recentrar cada frame. **Bloque G — cierre**. **Memoria de proyecto**: backlog anotado para preview 3D del mesh en el modal del picker (turntable rotation off-screen render — requiere FB nuevo, scope de hito futuro). **NO entra en F2H57**: snapshot undoable del paso convert, drag-and-drop SO→viewport (workflow Unreal — complementa el botón, futuro), plugin system para kits custom (Sub-fase 3+), templates compuestos de escena ("escena narrativa completa" — diferido), preview 3D en el picker. **Decisiones**: (1) **Workflow SFM-style sobre Hammer puro**: el dev confirmó que prefería ver los meshes ya importados antes que solo el file picker del SO. SFM cubre el caso común "ya tengo modelos, quiero reusarlos". El file picker queda como botón secundario dentro del mismo modal. (2) **Modal único de entry sin popup intermedio**: click directo en `+ Crear Entidad` abre el modal — un solo punto de entrada para crear (proyecto / archivo / vacía). Menos clicks que un popup de selección previa. (3) **Kits aditivos no-destructivos**: convertir agrega; quitar lo hace el dev manualmente vía Inspector. Más predecible que un modo destructivo con confirm. Snapshot undoable del convert diferido para no inflar v1. (4) **Demos removal minimal-risk**: solo menu entries; los helpers de DemoSpawners quedan como dead code. Hardening en hito futuro. (5) **Welcome ImGuiCond_Always**: el costo de recalcular center cada frame es despreciable y elimina la categoría completa de off-center bugs cuando la ventana cambia de tamaño. **Suite verde** (sin tests nuevos: workflow es UI visual).
