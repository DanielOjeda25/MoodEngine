## Qué entregó

### Bloque A — Schema `.moodshader` sobre `NodeGraph::Graph`

Wrapper sobre la infra `NodeGraph` reusada de F2H46 (Dialog Editor). 20 NodeKinds:

- Terminal: `OutputPBR` (5 inputs: albedo / metallic / roughness / normal / emissive).
- Generadores: `Color`, `Float`, `UV`, `Time`, `SampleTexture`.
- Aritméticos: `Add`, `Multiply`, `Lerp`, `Power`, `OneMinus`, `Saturate`, `Clamp`.
- Vectoriales: `Dot`, `Cross`, `Normalize`, `Length`, `Reflect`.
- PBR: `Fresnel`, `NormalMap`.

`createNode(kind, position)` popula sockets default + literals iniciales sensatos (Color blanco, Power.exp=2.0, Lerp.t=0.5, Fresnel.power=5.0). Serialización JSON aditiva.

### Bloque B — Generador GLSL

Walk topológico DFS post-order desde OutputPBR, emite SSA vars `v_<socketId>` con tipos GLSL correctos. Plantilla `shaders/pbr_graph_template.frag` (copia disciplinada de `pbr.frag` con markers `__SHADERGRAPH_DECLS__` + 5 `__SHADERGRAPH_<SLOT>__`). Detección de ciclos via DFS `visited` + `inStack`. Literals fallback cuando un input no está conectado.

### Bloque C — Editor visual `ShaderGraphEditorPanel`

Canvas con palette de 20 NodeKinds (right-click), inspector contextual del nodo seleccionado, zona Compile Output con warnings/errores. Toolbar New/Save/Save As/Close/Compile.

### Migración mid-hito: `imgui-node-editor` → `imnodes`

Wrapper `NodeGraphEditor` reescrito con imnodes (Nelarius, pin master `eb36902c`). Cero-touch al DialogEditorPanel. Disparado por bug de hit-test de proximidad upstream sin fix. Cita del dev: *"si quiero mover el nodo 2... estos bugs no me dejaron tranquilo"*.

### Bloque D — Integración MaterialAsset + Inspector dropdown estilo Blender

`MaterialAsset.shaderGraphPath` (vacío = PBR estándar). Inspector CollapsingHeader "Shader" con: scan del dir `assets/shaders/graphs/` per-frame + `ImGui::Combo` + item huérfano + botones "Editar" / "+ Nuevo".

### Bloque E — Compilación on-the-fly + cache + fallback al PBR

`ShaderGraphCache::getOrCompile(logicalPath, AssetManager) → IShader*` con pipeline `resolvePath → loadFromFile → loadDefaultTemplate → generateGlsl → hashGlsl → OpenGLShader(k_fromSource)`. Cache keyed por logical path + hash GLSL (recompila si difiere). Fallback transparente al PBR estándar si falla cualquier paso.

`OpenGLShader::FromSourceTag` constructor nuevo: compila vert desde archivo + frag desde string en memoria.

`RenderBatching`: materiales con `shaderGraphPath` no vacío caen a `nonBatchable` (path A.2 no-instanced). Razón: el path instanced usa `pbr_instanced.vert` que el cache no conoce.

## Bugs cazados durante validación + fixes

1. **`pbr_graph_template.frag` markers duplicados** en comentarios de cabecera Y en código → `replaceOnce` pisaba el comentario → GLSL inválido `ERROR: 0:9: '-' syntax error`. Fix triple: rename de markers en comentarios + post-substitution validation en generator + 2 tests regresión.
2. **`pfd::save_file` silent failure en Windows** → custom ImGui save modal como fallback + Ctrl+S directo + flash verde.
3. **Persistencia del shader graph al cambiar de panel** → same-path-open check + dirty check (nunca destruye silently).
4. **Delete keystroke para entities vs nodes** → gate por `nodeSelCount > 0 || linkSelCount > 0` en el wrapper imnodes.
5. **Tile delete dejaba "missing.png"** → `updateTileEntity` destruye entity cuando `tileAt(x,y) == Empty`; `deleteSelectedEntity` parsea `Tile_X_Y` y usa `SetTileCommand`.
6. **Node labels semánticos** → `createNode` setea `n->title = nodeKindDisplayName(kind)`.

## Decisiones clave (detalle en `DECISIONS.md`)

1. **`imnodes` sobre `imgui-node-editor`** — migración mid-hito por bug de hit-test de proximidad. Cero-touch al DialogEditorPanel.
2. **Solo fragment graph v1** — vertex sigue siendo `pbr.vert` estático.
3. **SSA naming `v_<socketId>`** — IDs únicos del NodeGraph garantizan unicidad sin tracking adicional.
4. **Markers con post-substitution validation** — error claro en vez de GLSL roto al driver.
5. **Materiales con shaderGraphPath caen a nonBatchable (A.2)** — el cache solo conoce `pbr.vert`.
6. **Cache keyed por logical path + hash GLSL** — recompila solo si el grafo cambia.
7. **Custom ImGui save modal como triple-fallback** sobre `pfd::save_file`.
8. **Dropdown estilo Blender en Inspector** sobre InputText manual.

## Limitaciones conocidas v1

- **Sin transparencia / alpha blending** — agendado como **F2H63** ("Materiales translúcidos").
- Sin vertex shader graph.
- Sin sub-graphs reutilizables.
- Sin uniforms custom desde el grafo.
- Sin compute shaders.
- `SampleTexture` v1 emite warning y usa `uAlbedoMap` fallback.
- `NormalMap` v1 emite `vWorldNormal` directo (sin TBN decode).
- Skeletal + shaderGraphPath cae al fallback PBR.
