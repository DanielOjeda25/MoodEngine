Bloques A-F, segundo hito de **Sub-fase 2.6 — Render polish**. Continúa el orden de impacto visual planeado en F2H55: bloom → AO → color grading → god rays. **Filosofía**: polish no-destructivo + engine-grade + industry-standard (no reinventar — algoritmo Crytek 2007 + normal reconstruido del depth, según pattern Filament/Godot 4). **Bloque A — 4 shaders nuevos**: `ssao.vert` (fullscreen triangle compartido, mismo patrón que bloom.vert / post_process.vert), `ssao.frag` (16 samples hemisferio en tangent-space rotados por IGN noise — Interleaved Gradient Noise de Jorge Jimenez COD AW 2014 — para romper patrón repetitivo; depth sampleado + reconstruct view-space position con inverse projection; normal vía `cross(dFdx, dFdy)` del view-space pos; range check con smoothstep para no contar oclusión entre objetos lejanos; bias 0.025 para evitar self-occlusion), `ssao_blur.frag` (4x4 box blur simple — bilateral con depth-aware weights queda para v2 si emerge calidad pobre), `ssao_composite.frag` (`hdr_color * ao_factor` modulando el HDR final). Portados de Filament (Apache 2.0) + Godot 4 (MIT). **Bloque B — `SSAOPass.{h,cpp}`** (`src/systems/render/`, ~150 líneas C++) con 2 FBs half-res LDR para AO factor (raw + blurred) + composite a FB HDR full-res. `apply(src, dst, proj, invProj, radius, intensity) → bool` mismo patrón que `BloomPass`. **`OpenGLFramebuffer` extendido**: en modo HDR el depth attachment ahora es **textura sampleable** (`GL_DEPTH24_STENCIL8` con `glTexImage2D` + `glFramebufferTexture2D`) en lugar de renderbuffer — necesario porque SSAO necesita leer depth desde un shader. LDR mantiene renderbuffer (más eficiente, no lo samplean los pixels del viewport final). Nuevo accesor `glDepthTextureId()`. **Wireup en `SceneRenderer::endFrame`**: orden ahora es scene HDR → **SSAO** → bloom → post-process. SSAO multiplica color HDR por AO factor `[0..1]` y escribe a `m_ssaoOutFb`; bloom lee del `m_ssaoOutFb` (no del `m_sceneFb` directo). Si SSAO está off o falla, `afterSsao` queda apuntando al `m_sceneFb` y el bloom pass lee de ahí — cero regresión vs F2H55 cerrado. `m_ssaoPass` + `m_ssaoOutFb` con try/catch silencioso en ctor (consistente con BloomPass). Resize del `m_ssaoOutFb` cableado en `renderScene` junto con sceneFb/bloomFb/viewportFb. **Bloque C — `EnvironmentComponent` extendido** con 3 campos: `ssaoEnabled` (master switch), `ssaoRadius` (0.5 default, view-space units; tipico 0.3-0.8), `ssaoIntensity` (1.0 default; 0=sin AO, 2=marcado). `SavedEnvironment` paralelo. `EntitySerializer` write aditivo (solo persiste si difiere del default — mismo patrón que bloom de F2H55 + ibl_intensity de Hito 18) + parse con defaults explícitos. `SceneLoader` copia Saved → Component. `SceneRenderer::applyEnvironmentFromScene` (que corre cada frame en `renderScene`) copia Component → cache → sliders propagan en vivo sin re-cargar mapa. **Bloque D — UI sliders**: sección "AO (sombras de rincón)" en `InspectorPanel_Light.cpp::renderEnvironmentSection` debajo de "Bloom (glow)" con checkbox `activar AO` + 2 sliders (`radio` 0.05-2.0, `intensidad AO` 0.0-3.0). Sliders ocultos si checkbox apagado. Undo records via `pushEditIfDone`. 4 i18n keys nuevas en es.json + en.json (`environment.ssao*`). **Bloque E — validación visual**: dev creó EnvironmentComponent vía menú demo + manipuló sliders. Cita verbatim: *"el SSAO funciona bien"*. Las esquinas y debajo de los objetos se oscurecen sutilmente al default; subiendo intensity a 3.0 el efecto se vuelve marcado y obvio. **Bloque F — cierre**. **Limitación conocida v1 documentada**: SSAO multiplica el color HDR FINAL, no solo el término ambient del PBR. Visualmente similar en la mayoría de escenas pero menos correcto físicamente (la luz directa también recibe AO). Refactor a integración con `pbr.frag` (modular solo el ambient) diferido a hito futuro si emerge calidad pobre. **NO entra en F2H56**: bilateral blur depth-aware (caja 4x4 alcanza), TAA para amortizar el noise inter-frame, HBAO/CACAO upgrade (port de Filament cubre el caso base — re-evaluar AMD CACAO si emerge presión de calidad), integración con PBR shader. **Decisiones**: (1) **Port de Filament/Godot vs FidelityFX CACAO**: portar pega con la filosofía no-reinventar + consistencia con cómo se hizo bloom F2H55. CACAO queda como upgrade futuro si la calidad pincha. (2) **Depth attachment como textura solo en HDR**: el LDR FB final (visible en ImGui) no necesita samplear depth — separación reduce memoria + mantiene path LDR optimizado. (3) **AO multiplicación al color final, no a ambient solamente**: trade-off de simplicidad v1 (no tocar PBR shader). Documentado como limitación conocida. (4) **Half-res AO buffer**: convención industria — 16 samples por pixel a full-res es caro y el blur 4x4 disimula la pérdida de resolución. (5) **Default ON con intensity=1.0**: mismo criterio que bloom — engine-grade provee defaults visuales sensatos; el dev apaga si quiere look plano. **Suite 926/9705 verde** (sin tests nuevos: SSAO requiere GL context como BloomPass; serialización roundtrip cubierta por el patrón existente).
